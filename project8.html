Nutrient Calculator Web Application

This opportunity from the USDA required a program that allows for food profile modifications and nutrient calculations. To tackle this challenge, my team developed a Flask web application that allows the user to search and select foods, dynamically build/modify food profiles, and calculate/display nutrient amounts calibrated per 100 grams. Our project was divided into two main versions. In the first version, I focused on developing nutrient calculations, data storage/retrieval methods, and saving/resetting capabilities. This version was a single page app with an accordion search/selection and tabs that allow the user to switch between the food profile and calculation display.

Despite the first version working fairly well, it did not live up to the professional quality and expectations that we had hoped for. For instance, every time the user selects or submits something, the whole page would refresh and therefore give users a choppy experience. That was when we decided to go for version 2, which completely restructured our codebase to make it more efficient and professional. It also included some features that were nonexistent in the first version, such as a login/admin system and upload/export data capabilities, not to mention a much smoother and professional user interface. We added an experienced team member to help us on this final version and I mainly worked on acting as a liaison between the stakeholders, which included the client, tester, supervisor, and our development team. I was also in charge of the final deployment and delivery to the client.
